# PIE database field reference

This doc describes the structure of the `pie_database.pkl` generated by `PIE.generate_database()`.

## Location
- Cache file: `data/PIE_dataset/data_cache/pie_database.pkl`

## Top-level structure
`db` is a nested dict:

- `db[set_id][video_id] -> video_record`

Examples:
- set ids: `set01` … `set06`
- video ids: `video_0001` … etc.

## Video record fields (`db[set_id][video_id]`)
Keys:
- `num_frames` (int): total frames in the clip
- `width` (int): video width (e.g., 1920)
- `height` (int): video height (e.g., 1080)
- `ped_annotations` (dict): pedestrian tracks, keyed by `ped_id`
- `traffic_annotations` (dict): traffic objects, keyed by `obj_id`
- `vehicle_annotations` (dict): per-frame vehicle sensor readings, keyed by `frame_id` (int)

---

## Pedestrian annotations (`ped_annotations[ped_id]`)
Keys:
- `frames` (list[int]): frame indices where this pedestrian appears
- `bbox` (list[list[float]]): bounding boxes `[xtl, ytl, xbr, ybr]` aligned with `frames`
- `occlusion` (list[int]): occlusion label aligned with `frames`
  - mapping: `none=0`, `part=1`, `full=2`
- `behavior` (dict[str, list[int]]): per-frame behavioral labels aligned with `frames`
  - `action`: `standing=0`, `walking=1`
  - `look`: `not-looking=0`, `looking=1`
  - `cross`: `not-crossing=0`, `crossing=1`, `crossing-irrelevant=-1`
  - `gesture`: `__undefined__=0`, `hand_ack=1`, `hand_yield=2`, `hand_rightofway=3`, `nod=4`, `other=5`
- `attributes` (dict[str, int|float]): static per-pedestrian metadata for this track  
  Observed keys:
  - `age`, `gender`, `intersection`, `num_lanes`, `signalized`, `traffic_direction`
  - `crossing_point`, `exp_start_point`, `critical_point`
  - `crossing`, `intention_prob`

### Alignment guarantee
For a pedestrian track, all per-frame lists align by index:
- `frames[i]` corresponds to `bbox[i]`, `occlusion[i]`, and each `behavior[*][i]`.

---

## Traffic annotations (`traffic_annotations[obj_id]`)
Keys:
- `frames` (list[int])
- `bbox` (list[list[float]]): `[xtl, ytl, xbr, ybr]`
- `occlusion` (list[int]): uses CVAT `occluded` flag for non-ped objects
- `obj_class` (str): traffic object category (e.g., `traffic_light`, `sign`, `vehicle`, `crosswalk`, etc.; depends on the video)
- `obj_type` (int|None): subtype id for classes like `traffic_light`, `sign`, `vehicle`; else `None`
- `state` (list[int]): meaningful for `traffic_light` only (same length as `frames`); otherwise typically empty  
  - mapping: `__undefined__=0`, `red=1`, `yellow=2`, `green=3`

---

## Vehicle annotations (`vehicle_annotations[frame_id]`)
- keyed by `frame_id` (int)
- value is a dict of sensor fields (float)

Observed keys:
- `GPS_speed`, `OBD_speed`
- `latitude`, `longitude`
- `heading_angle`
- `accX`, `accY`, `accZ`
- `gyroX`, `gyroY`, `gyroZ`
- `yaw`, `pitch`, `roll`
